# Vino 后端说明书

本文档面向 `backend/` 目录下的 Node.js 服务，详细拆解接口功能、权限、数据模型、与前端/小程序/后台的联动关系及测试方向。应用挂载在 `/api` 下，以下路径均相对 `/api`。

---

## 一、技术栈与整体结构

| 技术 | 说明 |
|------|------|
| Express | Web 框架 |
| Sequelize | ORM，连接 MySQL |
| JWT | 登录态，Bearer Token |
| Multer |  multipart 上传（头像、图片、Excel 等） |

**目录约定：**

- `src/routes/`：路由定义，挂载到主 router
- `src/controllers/`：业务逻辑
- `src/models/`：Sequelize 模型
- `src/middleware/auth.js`：认证与管理员校验
- `src/public/admin.html`：后台管理 SPA（与前端/小程序共用同一后端 API）

**主入口挂载（routes/index.js）：**

- `/auth` → authRoutes
- `/services` → serviceRoutes
- `/service-categories` → serviceCategoryRoutes
- `/orders` → orderRoutes
- `/addresses` → addressRoutes
- `/guides` → guideRoutes
- `/product-categories` → productCategoryRoutes
- `/home-config` → homeConfigRoutes
- `/messages` → messageRoutes
- `/inventory` → inventoryRoutes
- `POST /admin/generate-thumbs` → adminController.generateThumbs
- `POST /admin/seed` → seedController.seedData
- `GET /health` → 健康检查（匿名）

## 二、中间件与权限划分

### 2.1 authMiddleware

- **作用**：从请求头 `Authorization: Bearer <token>` 解析 JWT，校验通过则 `req.user = { id, username, role, ... }`。
- **失败**：返回 401，前端/小程序应清 token 并跳转登录。

### 2.2 adminMiddleware

- **作用**：在 auth 通过前提下，校验 `req.user.role === 'admin'`。
- **失败**：返回 403。

### 2.3 接口权限分类

| 类型 | 说明 | 典型接口 |
|------|------|----------|
| **公开** | 无需登录 | /health；/auth/send-code、send-sms-code、register、login、wx-login、alipay-login；/services、/services/:id；/guides/categories、/guides、/guides/:id；/home-config |
| **需登录** | authMiddleware | /auth/profile、bind-phone、my-products、bind-product、bind-by-qr-image、upload-avatar；/orders 创建/我的/详情/取消；/addresses 全量；/messages/mine、send、upload-image、unread |
| **需管理员** | authMiddleware + adminMiddleware | /auth/admin/users、admin/users/:userId/products/:productKey；/services、/service-categories 的增删改与 admin/list；/orders/admin/*；/guides/admin/*、/:id/qrcode；/product-categories 全量；/home-config 的增删改与 upload；/messages/admin/*；/inventory 全量；POST /admin/generate-thumbs、POST /admin/seed |

## 三、按模块拆解：功能点、入口、联动、测试

### 3.1 认证与用户（/auth）

| 方法 | 路径 | 中间件 | 控制器方法 | 功能说明 |
|------|------|--------|------------|----------|
| POST | /send-code | 无 | sendCode | 发送邮箱验证码（注册用） |
| POST | /send-sms-code | 无 | sendSmsCode | 发送短信验证码（注册/登录用） |
| POST | /register | 无 | register | 邮箱或手机号注册，成功后返回 token |
| POST | /login | 无 | login | 用户名密码或手机号+验证码登录 |
| POST | /bind-phone | auth | bindPhone | 已登录用户绑定手机号 |
| POST | /wx-login | 无 | wxLogin | 微信 code 换 openid，查/建用户，返回 token |
| POST | /alipay-login | 无 | alipayLogin | 支付宝 code 换 userId，查/建用户，返回 token |
| GET | /profile | auth | getProfile | 当前用户信息（不含密码） |
| PUT | /profile | auth | updateProfile | 更新昵称、头像 URL 等 |
| POST | /upload-avatar | auth + upload | uploadAvatar | 上传头像文件，写用户 avatar |
| GET | /admin/users | auth + admin | adminGetUsers | 用户列表分页（page、pageSize 默认 50）+ 统计 |
| DELETE | /admin/users/:userId/products/:productKey | auth + admin | adminUnbindProduct | 解除用户与某商品的绑定 |
| GET | /my-products | auth | myProducts | 当前用户已绑定商品列表（含指南信息） |
| POST | /bind-product | auth | bindProduct | 按序列号绑定商品（body.sn） |
| POST | /bind-by-qr-image | auth + upload | bindByQrImage | 上传二维码图片，解码 sn 后绑定 |

**功能入口：** 前端/小程序登录页、注册页、个人中心、我的产品、绑定产品页；后台「用户管理」页。

**前后端联动：** 前端/小程序登录后存 token，请求头带 Bearer；注册/登录返回 token 与 user；admin/users 仅后台调用，分页与统计供后台用户管理列表与统计卡片。

**测试方向：** 验证码发送与校验、重复邮箱/手机提示；登录/注册/第三方登录返回 token；profile/更新/头像上传；绑定与解绑产品、my-products 列表正确；admin 用户列表分页与统计、解绑后列表更新。

## 五、前后端联动关系小结

- **C 端（前端 Web + 小程序）**：使用公开接口 + 需登录接口；登录方式为账号密码/验证码（Web）或微信/支付宝 code（小程序）；不调用任何 admin 或 inventory 管理接口。
- **后台管理（admin.html）**：使用需管理员接口 + 部分需登录接口；与 C 端共用同一 baseUrl（如 `/api`），通过同一后端鉴权区分角色。
- **数据一致性**：用户、订单、地址、消息、绑定产品等均在同一个库中，C 端与后台读写同一批表，通过权限控制可见范围。

## 六、测试方向汇总（后端视角）

1. **鉴权**：无 token 访问需登录接口返回 401；非 admin 访问 admin 接口返回 403。
2. **参数与 body**：必填缺失返回 400；分页参数默认与上限（如 pageSize 最大 200）。
3. **业务规则**：订单状态流转、取消条件；地址唯一默认；绑定产品唯一性（一序列号一用户）；库存种类下有商品时不可删种类。
4. **分页**：mine 订单、admin 用户、admin 库存商品 等分页返回 list、total、page、pageSize 一致。
5. **文件上传**：头像、聊天图片、首页图片、指南文件、Excel 等大小与类型限制；返回可访问 URL 或正确错误信息。
6. **第三方**：微信/支付宝登录 code 换 openid/userId 与建号/登录逻辑；验证码发送与校验（邮箱/短信）。
7. **健康检查**：/health 无鉴权、返回固定结构。

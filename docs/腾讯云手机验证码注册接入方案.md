# 腾讯云手机验证码注册接入方案

## 一、概述

在现有「邮箱 + 验证码」注册基础上，增加「手机号 + 短信验证码」注册/登录能力，使用腾讯云短信（SMS）发送验证码，与现有用户体系兼容。

## 二、腾讯云侧准备

### 2.1 开通与配置

1. **开通腾讯云短信**
   - 登录 [腾讯云控制台](https://console.cloud.tencent.com/smsv2)
   - 开通「短信」产品，完成实名与资质（个人/企业）

2. **创建应用与密钥**
   - 在「访问管理」→「API 密钥」中创建子账号或使用主账号 SecretId/SecretKey
   - 建议为短信单独建子账号，权限仅勾选「短信」相关

3. **申请签名与正文模板**
   - **签名**：如「Vino服务」「Vino」等，用于短信开头
   - **正文模板**：验证码类模板，例如：
     - 内容：`{1}为您的验证码，请于{2}分钟内填写。如非本人操作，请忽略本短信。`
     - 变量：{1} 验证码，{2} 有效分钟数
   - 提交后等待审核通过，记下 **签名内容**、**模板 ID**

### 2.2 所需信息汇总

| 配置项       | 说明           | 示例/备注        |
|--------------|----------------|------------------|
| SecretId     | 腾讯云 API 密钥 | 在控制台获取     |
| SecretKey    | 腾讯云 API 密钥 | 同上             |
| SmsSdkAppId  | 短信应用 ID    | 在「应用管理」中 |
| SignName     | 已审核的签名   | 如「Vino服务」   |
| TemplateId   | 验证码模板 ID  | 如 1234567       |

## 三、后端改造

### 3.1 依赖

```bash
cd backend
npm install tencentcloud-sdk-nodejs
```

或仅安装短信子包（更轻量）：

```bash
npm install tencentcloud-sdk-nodejs-sms
```

### 3.2 配置

在 `backend/src/config/index.js`（或环境变量）中增加：

```js
// 腾讯云短信（可选，不配则仅保留邮箱注册）
sms: {
  secretId: process.env.TENCENT_SMS_SECRET_ID,
  secretKey: process.env.TENCENT_SMS_SECRET_KEY,
  smsSdkAppId: process.env.TENCENT_SMS_APP_ID,
  signName: process.env.TENCENT_SMS_SIGN_NAME || 'Vino服务',
  templateId: process.env.TENCENT_SMS_TEMPLATE_ID, // 验证码模板 ID
  codeExpireMinutes: 5,
},
```

生产环境建议用环境变量，不要把 SecretKey 写进代码库。

### 3.3 验证码存储

- 与现有邮箱验证码类似：内存或 Redis 存 `phone -> { code, expireAt }`。
- 若当前邮箱验证码用内存，可复用同一结构，key 改为 `sms:${phone}` 或 `email:${email}` 区分。
- 建议：**同一接口**，通过参数区分是发邮箱还是发短信（见下）。

### 3.4 接口设计

**方案 A：与现有邮箱共用「发验证码」接口（推荐）**

- 现有：`POST /auth/send-code`，body `{ email }`。
- 扩展为：body 支持 `{ email }` 或 `{ phone }`，二选一。
  - 传 `email`：走现有邮箱逻辑，校验邮箱格式、是否已注册等。
  - 传 `phone`：校验手机号格式、是否已注册（若需限制同一手机只能注册一次）、调用腾讯云短信发送验证码并写入存储。

**方案 B：单独手机验证码接口**

- 新增：`POST /auth/send-sms-code`，body `{ phone }`。
- 校验 11 位大陆手机号、是否已注册等，发短信并写入验证码。

**注册接口扩展**

- 现有：`POST /auth/register`，body `{ username, password, email, code, nickname? }`。
- 扩展为支持手机号注册，例如：
  - body 增加可选 `phone`，当传了 `phone` 时，必传 `smsCode`（或复用 `code` 字段，通过是否同时传 `email` 判断是邮箱码还是短信码）。
  - 校验逻辑：若为手机号注册，则用 `phone + smsCode` 校验；若为邮箱注册，则用 `email + code` 校验。
  - 用户表需有 `phone` 字段（若尚无）：`User` 表增加 `phone`（唯一、可空），注册时写入。

### 3.5 腾讯云短信发送示例（Node.js）

```js
const tencentcloud = require('tencentcloud-sdk-nodejs');
const SmsClient = tencentcloud.sms.v20210111.Client;

const client = new SmsClient({
  credential: {
    secretId: config.sms.secretId,
    secretKey: config.sms.secretKey,
  },
  region: 'ap-guangzhou',
  profile: { httpProfile: { endpoint: 'sms.tencentcloudapi.com' } },
});

// 发送验证码
const code = String(Math.floor(100000 + Math.random() * 900000));
await client.SendSms({
  SmsSdkAppId: config.sms.smsSdkAppId,
  SignName: config.sms.signName,
  TemplateId: config.sms.templateId,
  PhoneNumberSet: ['+86' + phone],
  TemplateParamSet: [code, String(config.sms.codeExpireMinutes)],
});
// 将 code 与过期时间写入存储，key 为 phone
```

发送成功后，将 `code` 与过期时间写入验证码存储，校验时用 `phone` 取码比对。

### 3.6 安全与限流

- **频率限制**：同一手机号 60 秒内只能发 1 次；同一 IP 每分钟/每小时限制条数（如 10 条/小时）。
- **验证码**：6 位数字，5 分钟内有效，使用一次即失效。
- **防刷**：前端可加图形验证码或滑块（可选），后端必做频率与参数校验。

## 四、前端改造

### 4.1 注册页

- 增加「手机号注册」入口或 Tab（与邮箱注册并列）。
- 表单项：
  - 手机号（11 位）
  - 短信验证码（输入框 + 「获取验证码」按钮，带 60 秒倒计时）
  - 用户名、密码、昵称（可选）等与现有一致。
- 提交时：传 `phone` + `smsCode`（或 `code`），不传 `email`，走手机号注册逻辑。

### 4.2 登录页（若支持手机号登录）

- 可选：增加「手机号 + 验证码」登录 Tab。
- 接口：若后端提供「手机号 + 验证码」登录接口，前端调该接口完成登录并写入 token。

## 五、数据库

- **User 表**：若尚无 `phone` 字段，增加 `phone`（VARCHAR(20)，唯一索引，可空），便于手机号注册/绑定与后续登录。

## 六、实施步骤建议

1. 腾讯云侧：申请签名与验证码模板，审核通过后拿到 TemplateId、SignName、SmsSdkAppId 及密钥。
2. 后端：安装 SDK、增加配置与短信发送工具函数、验证码存储（按 phone 存）。
3. 后端：扩展 `send-code` 或新增 `send-sms-code`，实现发码与限流。
4. 后端：扩展 `register` 支持 `phone + smsCode`，并增加/使用 `User.phone`。
5. 前端：注册页增加手机号+短信验证码表单与调用逻辑。
6. 联调与测试：真机收码、注册成功、登录正常。
7. 上线：配置生产环境环境变量，关闭或限制调试发码。

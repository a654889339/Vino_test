# Vino 前端（Web）说明书

本文档面向 `frontend/` 目录下的 Vue 3 网页端，详细拆解功能点、入口、测试方向及与后端的联动关系。

---

## 一、技术栈与目录结构

| 技术 | 说明 |
|------|------|
| Vue 3 | Composition API + `<script setup>` |
| Vue Router | 单页路由，history 模式 |
| Vant 4 | 移动端 UI 组件 |
| Pinia | 用户状态（token、userInfo） |
| Axios | 请求封装，baseURL 为 `/api` |

**主要目录：**

- `src/views/`：页面组件
- `src/router/index.js`：路由配置
- `src/api/`：`request.js`（axios 实例）、`index.js`（各业务 API）
- `src/stores/user.js`：登录态与用户信息
- `src/components/`：公共组件（如 ChatWidget、LodImg、SplashScreen）

**存储约定：**

- `localStorage['vino_token']`：登录 Token，请求头自动带 `Authorization: Bearer <token>`
- `sessionStorage['vino_splash_shown']`：闪屏是否已展示（仅展示一次）

## 二、功能点详细拆解

### 2.1 首页（Home）

| 项目 | 说明 |
|------|------|
| **路由** | `/`，组件 `Home.vue` |
| **功能点** | 1）展示首页配置（开场动画、首页 Logo、背景图、导航区块、热门服务、我的商品入口等）；2）分享二维码；3）点击导航/服务/商品跳转对应页 |
| **数据来源** | `GET /api/home-config`（无登录），按 `section` 区分：splash、headerLogo、background、navSection、hotService、recommend 等 |
| **功能入口** | 地址栏输入 `/` 或从任意页点击「首页」 |
| **前后端联动** | 前端拉取 home-config 列表，按 section 渲染；Logo/背景等图片使用返回的 `imageUrl` |
| **测试方向** | 未配置时占位/默认展示；配置多条背景时轮播；未登录与已登录下首页均能打开；分享二维码可生成且链接正确 |

### 2.5 我的订单（Orders）

| 项目 | 说明 |
|------|------|

## 四、测试方向汇总

1. **未登录**：需登录页均能正确跳转登录；登录后回到原流程或首页。
2. **登录态**：token 过期或 401 后清空并跳转登录。
3. **首页**：配置为空/部分配置时的展示；Logo、背景与后台「首页动画配置」一致。
4. **订单**：分页加载更多、Tab 切换、取消订单后列表更新。
5. **地址**：增删改查与默认切换；与下单选地址联动。
6. **绑定产品**：URL 参数 sn、手动输入、二维码上传三种方式；重复绑定、他人已绑定提示。
7. **注册**：邮箱/手机双通道、验证码、Logo 展示。
8. **客服**：发消息、收消息、未读数与后台/小程序一致。

## 五、与后端联动关系小结

- **公开接口（不传 token）**：首页配置、服务列表/详情、指南分类/列表/详情、登录、注册、验证码。
- **需登录接口（传 token）**：profile、地址、订单创建/我的/取消、我的产品、绑定产品、消息、上传头像/图片。
- **前端不直接调用的后端能力**：管理端（订单管理、用户管理、库存、首页配置等）由后台 SPA（admin.html）调用，与前端共用同一后端 API 基路径（如 `/api`）。

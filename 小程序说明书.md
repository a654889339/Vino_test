# Vino 小程序说明书

本文档覆盖 **微信小程序**（`wechat-mp/`）与 **支付宝小程序**（`alipay-mp/`）两套原生项目，说明功能点、入口、测试方向及与后端的联动关系。两端业务逻辑与后端接口一致，仅运行环境与 API 调用方式不同。

---

## 一、项目结构与环境差异

| 项目 | 目录 | 语法/扩展名 | 入口 |
|------|------|-------------|------|
| 微信小程序 | `wechat-mp/` | wx.* API；.wxml / .wxss / .js / .json | app.js、app.json |
| 支付宝小程序 | `alipay-mp/` | my.* API；.axml / .acss / .js / .json | app.js、app.json |

**共同约定：**

- 后端 baseUrl：`http://106.54.50.88:5202/api`（与前端 Web、后台管理共用）
- 登录态存储 key：`vino_token`
- 请求成功判断：`res.data.code === 0`；401 时清除 token 并提示登录

**请求封装：** 均在 `app.js` 的 `request(options)` 中统一封装，url 为 `baseUrl + options.url`，自动带 `Authorization: Bearer <token>`；各页面通过 `getApp().request({ url, method, data })` 调用后端。

## 三、功能点详细拆解

### 3.1 首页（pages/index/index）

| 项目 | 说明 |
|------|------|
| **功能点** | 展示首页配置（Logo、背景、导航区块、热门服务、我的商品入口等）；点击跳转对应列表/详情 |
| **后端接口** | `GET /home-config`（无 token） |
| **前后端联动** | 与 Web 前端一致，按 section 解析并渲染；Logo/背景使用后台「首页动画配置」 |
| **测试方向** | 首屏加载、配置为空时的表现；各入口跳转正确；与 Web 首页展示一致（除样式外） |

### 3.7 登录（pages/login/login）

| 项目 | 说明 |
|------|------|
| **功能点** | **微信**：wx.login 取 code，调用 `POST /auth/wx-login` 传 code；**支付宝**：取得 authCode，调用 `POST /auth/alipay-login` 传 code。成功后 setToken、拉 profile、跳转 |

## 四、与后端接口对照（小程序侧）

小程序通过 `getApp().request({ url, method, data })` 调用，url 为相对路径（如 `/auth/profile`），实际请求为 `baseUrl + url`。

| 功能 | 方法 | 后端路径（相对 /api） | 说明 |
|------|------|------------------------|------|
| 登录（微信） | POST | /auth/wx-login | body: { code } |
| 登录（支付宝） | POST | /auth/alipay-login | body: { code } |
| 用户信息 | GET | /auth/profile | 需 token |
| 更新资料 | PUT | /auth/profile | 需 token |
| 上传头像 | POST | /auth/upload-avatar | multipart，需 token |
| 我的产品 | GET | /auth/my-products | 需 token |
| 绑定产品 | POST | /auth/bind-product | body: { sn }，需 token |
| 服务列表 | GET | /services | 公开 |
| 服务详情 | GET | /services/:id | 公开 |
| 创建订单 | POST | /orders | body 同 Web，需 token |
| 我的订单 | GET | /orders/mine | 分页参数，需 token |
| 取消订单 | PUT | /orders/:id/cancel | 需 token |
| 指南分类 | GET | /guides/categories | 公开 |
| 指南列表 | GET | /guides | 公开 |
| 指南详情 | GET | /guides/:id | 公开 |
| 首页配置 | GET | /home-config | 公开 |
| 地址列表 | GET | /addresses | 需 token |
| 地址增删改 | POST/PUT/DELETE | /addresses、/addresses/:id | 需 token |
| 设默认地址 | PUT | /addresses/:id/default | 需 token |
| 消息列表/发送/未读 | GET/POST | /messages/mine、/messages/send、/messages/unread | 需 token |
| 上传消息图片 | POST | /messages/upload-image | multipart，需 token |

## 五、测试方向汇总

1. **双端一致性**：同一账号在微信与支付宝小程序下，订单、地址、我的产品、消息等数据一致，且与 Web、后台一致。
2. **登录**：微信 code / 支付宝 authCode 换 token；首次建号、再次登录；token 持久化与 401 清除。
3. **权限**：需登录页在未登录时引导登录（checkLogin）；401 后清 token 并提示。
4. **首页**：配置与后台一致；Logo/背景来自「首页动画配置」。
5. **订单**：分页、Tab、取消与后端一致。
6. **地址**：增删改查、默认与下单选地址联动。
7. **客服**：与 Web/后台消息互通；未读数正确。
8. **域名与请求**：baseUrl 可配置（如正式环境）；微信/支付宝 request 域名白名单需包含后端域名。

## 六、与前端 Web、后端的联动关系

- **共用后端**：小程序与 Web 前端、后台管理（admin）共用同一后端 API（baseUrl 一致）；数据（用户、订单、地址、消息、绑定产品等）互通。
- **登录方式**：Web 为账号密码或验证码；小程序为微信/支付宝第三方登录（wx-login、alipay-login），同一后端用户表，通过 openid/alipayId 关联。
- **功能对应**：小程序与 Web 功能一一对应（首页、产品、服务、订单、我的、地址、资料、我的产品、客服、绑定产品等），仅 UI 与交互按平台实现；接口路径与参数与《前端说明书》中 API 对照表一致（登录接口除外）。
- **后台管理**：管理员在 Web 后台管理订单、用户、库存、首页配置、消息回复等；小程序端仅使用用户端接口，不调用管理端接口。
